1. Historia tecnica do caso

O caso Citi Bike usa dados publicos reais para estudar viagens, e-bikes, estações e pressão operacional. A arquitetura e semelhante ao Retail, mas a origem e o problema sao reais e urbanos.

FonteTrip history e GBFS oficiais do Citi Bike.
StagingSQLite guarda dados crus organizados localmente.
OLAPDuckDB cria fact tables e dimensoes analiticas.
dbtVersiona staging, marts e testes de qualidade.
MLClassifica uso de e-bike e agrupa perfis de viagem.
StreamlitMostra KPIs operacionais e leitura rapida do sistema.

2. Matriz completa: codigo, teoria, input e output

Ficheiro / pasta Teoria aplicada Input Output Como explicar
python/download_citibike_data.py Ingestao de dados publicos e APIs/feed files. Citi Bike trip history e GBFS. data/raw/tripdata_* e data/raw/gbfs/*.json. Busca a materia-prima real do sistema antes de qualquer transformacao.
python/setup_sqlite.py + sql/sqlite/ Staging relacional e normalizacao minima. CSVs e JSONs descarregados. data/processed/citibike_oltp.db. Cria uma base operacional local para organizar os dados crus.
python/setup_duckdb.py + sql/duckdb/ OLAP, fact table e dimensoes. SQLite/staging processado. data/processed/citibike_analytics.duckdb. Converte viagens e estações em tabelas prontas para analise.
dbt_citibike/models/staging/ Staging governado: casting e contratos de campos. Tabelas do DuckDB. stg_trips, stg_stations, stg_station_status. Padroniza as fontes antes de criar modelos finais.
dbt_citibike/models/marts/ Modelacao analitica para BI/ML. Modelos staging. fct_trips, dim_stations. Entrega entidades analiticas claras: viagens e estacoes.
dbt_citibike/models/schema.yml Testes de qualidade e relacoes. Modelos dbt. Testes not_null, unique e relationships. Mostra governanca: os modelos têm regras verificaveis.
python/ml_citibike.py Classificacao supervisionada e clustering nao supervisionado. data/processed/citibike_analytics.duckdb. citibike_ml_metrics.json, ebike_feature_importance.csv, trip_profile_clusters.csv. Usa viagens para prever e-bike e descobrir perfis de viagem.
streamlit/app.py Visual analytics operacional. DuckDB analitico e outputs de ML. Dashboard local em browser. Transforma tabelas em leitura sobre procura, e-bikes e estacoes.
python/validate_project.py Validacao end-to-end reproduzivel. Codigo do repositorio. Pipeline sample, DuckDB validado, ML gerado e Streamlit compilado. Confirma que o caso continua executavel numa maquina limpa.

3. Outputs reais e interpretacao

3.1 Dados e bases geradas

  • data/raw/tripdata_csv/: viagens oficiais descarregadas.
  • data/raw/gbfs/: snapshots de estado do sistema.
  • data/processed/citibike_oltp.db: staging operacional SQLite.
  • data/processed/citibike_analytics.duckdb: camada analitica DuckDB.

3.2 Metricas de ML geradas

{
  "classification_ebike_usage": {
    "accuracy": 0.7244
  },
  "cluster_count": 4
}

Leitura: o modelo acerta cerca de 72% na classificacao de uso de e-bike com features simples como estacao, hora, duracao, dia da semana e tipo de utilizador.

3.3 Importancia das features

start_station_encoded -> 0.3280
end_station_encoded   -> 0.2590
start_hour            -> 0.1628
trip_minutes          -> 0.1619
start_day_of_week     -> 0.0814
member_encoded        -> 0.0069

Leitura: as estacoes de origem/destino pesam muito, o que sugere que a disponibilidade e o contexto geografico influenciam o uso de e-bike. Hora e duracao tambem ajudam a separar comportamentos.

3.4 Clusters de perfis de viagem

cluster 0 -> viagens casuais, media 15.8 min, alta proporcao de e-bike
cluster 1 -> membros, media 10.0 min, e-bike
cluster 2 -> membros, media 10.2 min, bicicleta classica
cluster 3 -> viagens muito longas/outliers, media 904.5 min

Leitura: os clusters ajudam a contar a historia operacional. Alguns grupos parecem deslocacoes curtas de membros, outros parecem uso casual ou outliers que exigem limpeza/analise mais cuidada.

Frase para apresentacao: "No Citi Bike, os outputs mostram uma pipeline com dados reais: download, staging, camada analitica, dbt, ML e dashboard. O modelo sugere que geografia, hora e duracao explicam parte relevante da escolha de e-bike."
