Citi Bike: ML, Comparacao, Visualizacoes e Dados de 2 Anos

Guia para expandir o caso Citi Bike com mais experiencias de Machine Learning, validacao comparavel, visualizacoes e download dos ultimos dois anos oficiais.

Hub · Codigo + outputs · Uniformizacao

1. Descarregar os ultimos 2 anos

O script agora aceita --years 2. Em 28/04/2026, a consulta ao bucket oficial seleciona 24 meses completos, de 202404 a 202603, com 48 archives no total porque existem ficheiros Citi Bike NYC e JC. Como os ZIPs podem ser grandes, valida primeiro com amostragem na ingestao.

python python/run_pipeline.py --years 2 --sample-rows-per-file 20000
python python/ml_citibike.py
streamlit run streamlit/app.py

Para ingerir tudo depois de confirmares espaco e tempo:

python python/run_pipeline.py --years 2
python python/ml_citibike.py
Modo recomendado para poupar disco: usa sempre --sample-rows-per-file. Nesse modo, o script descarrega cada ZIP oficial, extrai apenas a amostra configurada e apaga o ZIP logo de seguida. Assim consegues cobrir os 24 meses sem deixar dezenas de GB em data/raw/.

2. Casos de ML que podes testar e comparar

CasoObjetivoModelos candidatosMetricasVisualizacao
Classificar uso de e-bike Prever se uma viagem usa bicicleta eletrica. Logistic Regression, Random Forest, Gradient Boosting, XGBoost/LightGBM. Accuracy, ROC AUC, F1, confusion matrix. Feature importance, ROC, matriz de confusao.
Prever duracao da viagem Estimar trip_minutes. Linear Regression, Random Forest Regressor, Gradient Boosting. MAE, RMSE, R2, erro por hora/estacao. Real vs previsto, residuos por hora.
Clusters de viagens Descobrir perfis de utilizacao. K-Means, Gaussian Mixture, HDBSCAN/DBSCAN. Silhouette, Davies-Bouldin, estabilidade temporal. Scatter hora vs duracao, mapa por estacao.
Prever member vs casual Classificar tipo de utilizador. Logistic Regression, Random Forest, Gradient Boosting. F1 macro, ROC AUC, recall por classe. Heatmap hora/dia por tipo de utilizador.
Forecast de procura Prever numero de viagens por hora/dia. Moving average, Exponential Smoothing, ARIMA/Prophet. MAE temporal, MAPE, backtesting. Linha historico vs forecast por hora/dia.
Risco de estacao Identificar estações perto de vazio/saturacao. Rules baseline, Isolation Forest, classificacao se houver labels. Taxa de alerta, cobertura, inspecao operacional. Ranking de utilizacao, mapa, barras por disponibilidade.

4. Como validar e comparar

python python/run_pipeline.py --years 2 --sample-rows-per-file 20000
python python/ml_citibike.py
python python/validate_project.py
streamlit run streamlit/app.py

Compara modelos sempre com a mesma janela de meses. Para validação mais honesta, evita dividir aleatoriamente quando estiveres a fazer forecasting: usa treino em meses antigos e teste nos meses mais recentes.
